# 战区数据展示网页 - 设计规格

## 概述

一个用于展示战双帕弥什战区数据的网页，包含当前战区信息和全服排行榜。

## 功能需求

### 1. 页面标题
- 显示服务器名称（CN）
- 参与人数
- 更新时间

### 2. 战区卡片区域
横向排列3个战区卡片，每个卡片包含：
- 战区名称
- 怪物刷新规则
- 天气效果（名称+描述）
- 增益效果（名称+描述）

**混合区处理**：熵钟异数用外框包裹，内含两个独立子区卡片（镭射合金、蚀刃峰岭），每个子区有自己的怪物、天气、增益。

### 3. 排行榜
- 显示TOP 100玩家
- 每行显示：排名、玩家名、公会名、选择的战区、分数
- 战区名称用颜色标签区分

## 数据来源

API: `https://api.huaxu.app/servers/cn/warzone/current/16`

### 数据结构

```json
{
  "status": "success",
  "data": {
    "warzone": {
      "server": "cn",
      "activity": 568,
      "challenge": 16,
      "start": "2026-04-26T21:00:00Z",
      "end": "2026-05-03T10:00:00Z",
      "members": 6060,
      "updatedAt": "2026-05-03T19:31:00.000925Z",
      "area": {
        "weather": {...},
        "zones": [...]
      },
      "rankings": [...]
    }
  }
}
```

### zones 数组结构
每个zone包含：
- `id`: 战区ID
- `name`: 战区名称
- `description`: 怪物刷新规则描述
- `buffDescription`: 增益描述
- `element`: 元素类型
- `buffs[]`: 增益列表（id, name, description）
- `weathers[]`: 天气列表（name, description, icon, background）

### rankings 数组结构
每个排名包含：
- `rank`: 排名
- `player`: 玩家信息（id, name, sign, portrait, frame, guildId, guildName）
- `score`: 总分
- `zones[]`: 各战区得分（id, score, environment, characters[]）

## 设计风格

- 深色主题
- 科技感
- 类似战双官网风格
- 渐变背景
- 霓虹蓝点缀

## 技术栈

- 纯HTML/CSS/JavaScript
- 直接调用API获取数据
- 响应式设计
